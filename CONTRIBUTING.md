# Contribuir a PatrocinApp

1. Partir de `desarrollo` y crear una rama en español (`funcionalidad/`, `datos/`, `pruebas/`, `diseno/` o `infraestructura/`).
2. No versionar credenciales, tokens, bases, binarios de evidencias ni archivos `.env`.
3. Ejecutar `./mvnw clean verify` antes de solicitar integración.
4. Abrir un Pull Request real hacia `desarrollo`, completar la plantilla y adjuntar pruebas reproducibles.
5. Solicitar revisión a una persona distinta del autor. No se atribuyen revisiones ni coautorías que no hayan ocurrido.
6. La promoción de `desarrollo` a la rama principal exige validación funcional, CI verde y aprobación humana.

Convenciones de commit: `feat`, `fix`, `test`, `docs`, `refactor` y `chore`, seguidas de una descripción breve en español.
